## 2. How to Contribute

### Reporting Bugs
- Check the page to see if the bug has already been reported.
- Use the "Bug Report" template if available.
- Include steps to reproduce, the expected behavior, and your environment details.

### Suggesting Enhancements
- Open an Issue to discuss the enhancement before starting work.
- Explain the "Why" behind the feature and how it fits Precog Markets roadmap.

### Pull Request Process
- **Fork** the repository and create your branch from master.
- **Lint & Test**: Ensure your code follows our style guide and passes all existing tests.
- **Sign Your Commits**: Use `git commit -s` to sign off on your changes (Developer Certificate of Origin).
- **Documentation**: Update the `README.md` or internal docs if you change any public-facing logic.
Open PR: Fill out the PR template completely. A maintainer will review your code shortly.
<hr/>

## 3. Development Guidelines
- **Smart Contracts**: We use Hardhat for testing. Please ensure 100% branch coverage for new logic and that all tests pass ok.
- **Security**: If you find a critical vulnerability, **do not** open a public issue. Please email **admin@precog.markets** instead.
- **Style**: Follow the Official Solidity Style Guide.
<hr/>

## 4. Community & Code of Conduct
We are committed to fostering a welcoming and harassment-free environment. Above all else, please be respectful of the people behind the code. Any kind of aggressive or disrespectful comments, issues, and language will be removed.

Issues and PRs that are obviously spam and unhelpful to the development process or unrelated to the core code will also be closed.
